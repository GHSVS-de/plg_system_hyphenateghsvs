Don't forget to save the plugin after configuring the languages.

Many languages are preconfigured here as examples but are deactivated. You can remove (red button) or add (green) or order (blue) languages.

The Hyphenopoly scripts check if a browser supports hyphenation on its own for the here configured AND ACTIVATED languages. Only if NOT then Hyphenopoly will try to load a so called "pattern" for each unsupported language. These are files with file extensions *.hpb, e.g. en-gb.hpb. You can see all available language patterns in the dropdown lists of column Language (Hyphenopoly).

Normally a Joomla page has at least 1 lang attribute, namely inside the html tag, e.g. in a german page <html lang="de-de" dir="ltr">.

If you run a multilingual Joomla site with a language switcher German/English there are also english pages with <html lang="en-gb" dir="ltr">.

That means: You must configure both languages in the plugin because they have different hyphenation rules.

Let's start with the easier one, english:

  • The lang attribute is en-gb and there's a pattern en-gb in the dropdown list in column Language (Hyphenopoly). Perfect! Select it and leave the field of column Lang Tag empty.
  • Enter a single and at least 12 characters long and polysyllabic word in column Word . For english it could be "hyphenationalgorithm".
  • Don't forget to activate this language row in column Active.

Let's now configure german:

  • The lang attribute inside page HTML is de-de and, bad news, there is NO pattern de-de yet (while I write this description in 2019) in the dropdown list in column Language (Hyphenopoly).
  • Try to find another similiar pattern and select it. I propose de.
  • In column Lang Tag enter de-de. Thus Hyphenopoly knows that it shall fall back to pattern de if it finds a lang=de-de in the page.
  • Enter a single and at least 12 characters long and polysyllabic word in column Word . For german it could be "Silbentrennungsalgorithmus".
  • Activate the language row in column Active.

The tabulator Cleanup&Debug&Log tells you some stories about messages that help you to correct your language settings.

Keep in mind:

Any here configured and activated language will be checked by Hyphenopoly. You should not activate unnecessary languages. Any unnecessary check eats up some performance of your site.

Experts:

The Word FORCEHYPHENOPOLY will force Hyphenopoly to load a language pattern even if the browser doesn't need it. Note: Disable CSS-hyphenation while using "FORCEHYPHENOPOLY" for testing purposes.
