Whenever Hyphenopoly finds

  1. a lang tag inside the page HTML that is not configured (= needed language pattern not loaded)
  2. a configured language that is not needed in the page (= unneccessary language pattern loaded)

it throws an unproblematic informatory error/warning/info inside the console of your browser. Guests of your page won't see them and Hyphonopoly will not stop working but perhaps doesn't hyphenate correctly.

E.g. messages like

  1. Element with 'en-gb' found, but 'en-gb.hpb' not loaded. Check language tags!
  2. engine for language 'fr' loaded, but no elements found.

That can be very helpful to debug your plugin configuration.

Messages of type 1. tell you that you haven't configured a needed language for a lang tag en-GB. Solution: Configure it correctly in the plugin.

Messages of type 2. will occur nearly on every Joomla page if you run a multilingual site even if you have configured the plugin correctly for all languages. It's clear that a german page doesn't contain english elements marked with a lang=en-gb in most cases.

If you don't need/want these messages in the browser console set NO here.
