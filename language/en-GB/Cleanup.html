Be aware: I cannot guarantee that the Cleanup feature will lead to a more performant page in any circumstance. Therefore I mark this setting "experimental".

Let's say you have a multilingual Joomla site (de-de and en-gb) with a language switcher. Because Hyphonopoly doesn't check if the languages that you've configured for all of your pages are really needed by the displayed page you will see error/warning/info messages in the browser console, e.g. for the german page: engine for language 'en-gb' loaded, but no elements found. (= I couldn't find a lang=en-gb in current HTML code).

Even if these messages are unproblematic and can help you a lot to configure the plugin correctly it's impossible to get rid of them by correct language configuration for a multilingual site.

Hyphenopoly won't stop working but it's a performance issue when Hyphenopoly loads too much not needed language patterns.

If you activate Cleanup the plugin tries to remove unneccessary languages from the Hyphenopoly configuration script before loading it:

  • Option YES: If Cleanup detects that none of the configured languages are needed on a page it will leave everything unchanged (= like configured by you) to give you a chance to debug the errors with console messages of Hyphenopoly.
  • If you select option FORCE the cleanup will remove the whole configuration script in that case which means Hyphenopoly scripts won't be loaded. That means also: No console messages of Hyphenopoly.

The scripts that set the hyphenate and donthyphenate classes stay always untouched.

See also related setting Disable Console Messages.
