The CSS selectors that you enter in this field will get a CSS class hyphenate. Thus the content of the marked HTML element will be hyphenated and it's child elements. The markup is the same like in a CSS file. You can enter HTML tags or classes (with a leading dot) or ids (with a leading #) or combinations.

Example with tag body

Nearly the whole page will be hyphenated becaus the <body>-Tag of your page gets a class hyphenate => <body class="hyphenate"> and because all child elements of a tagged element will inherit the behavior. Normally you must exclude some of the child elements of body with the setting Do-Not-Hyphenate-Selectors then to stop the hyphenation for them.

Think before you use the body tag! It's not recommended to set a tag body here if you want to avoid performance issues. You should define smaller containers here that shall be hyphenated. In Joomla that's normally only the content part (output of components like articles).

Tipp for the lazy ones:

  • Surround <jdoc:include type="component" /> in template-index.php by an additional div container. 
    <div id="div4component">
<jdoc:include type="component" />
</div>
  • Then configure #div4component as Hyphenate-Selector.
Example with CSS class .item-page

All Tags with a class item-page will get an additional class hyphenate. <div class="item-page something-else"> => <div class="item-page something-else hyphenate">

Example with CSS id #icons.small

A <div id="icons" class="small some-other-class"> will become <div id="icons" class="small some-other-class hyphenate">

Enter 1 selector per line. Like this:

.item-page
.item p
#icons.small
div.container
section.somewhere div.inside
#myid .outside
[itemprop="headline"]
[itemprop="articleBody"]

If you leave this field empty and field Do-Not-Hyphenate-Selectors is empty, too, the plugin will NOT load the hyphenation scripts. Enter a nonsense selector like "nonsensexyzzz" if you want to force loading, e.g. if you prefer to set the hyphenate and donthyphenate classes manually in your HTML markup.
